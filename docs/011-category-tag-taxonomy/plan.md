# 分類與標籤整理規劃

## 背景

目前商品 taxonomy 使用單一 `category_id` 作為主分類，並以 `tag_ids` 補充可多重套用的交叉語意。既有分類為「居家、廚房、電腦、3C、影音、食材、其他」，在衛浴、電源、清潔、影音設備、生活百貨等商品上容易出現分類語意重疊或硬塞問題。

本次規劃目標是重新整理主分類與 tag：主分類作為網站導覽入口；tag 同時承擔兩種語意——category 內的細分（subcategory，方便瀏覽與尋找商品）與跨 category 的交叉篩選。原則上不為單一商品建立專用 tag。

## 目前收斂方向

## 共用 Tag 原則

以下 tag 可跨 category 共用，不需要為不同 category 建立重複版本：

| Tag | 可跨用於 |
| --- | --- |
| 顯示器 | 電腦3C、影音劇院 |
| 電源充電 | 電腦3C、網路通訊、小家電 |
| 線材轉接 | 電腦3C、影音劇院、網路通訊 |
| 清潔 | 小家電、生活百貨 |
| 衛浴 | 小家電、生活百貨 |
| 廚房 | 小家電、大家電、生活百貨 |
| 照明 | 小家電、生活百貨 |
| 固定安裝 | 大家電、生活百貨中的衛浴設備 |

「廚房」不設主分類，導覽動線改由共用 tag 承擔（既有 kitchen 商品會分散到小家電、大家電、生活百貨）。

## 已知 Mapping 判斷

| 商品群 | 主分類方向 | 補充 tag 方向 |
| --- | --- | --- |
| 行動電源、充電器、電池、UPS | 電腦3C | 電源充電 |
| Unifi、Zyxel 等網通設備 | 網路通訊 | 網路設備 |
| 電視、Apple TV、HomePod、Soundbar、喇叭、耳機、HDMI 線 | 影音劇院 | 顯示器、訊源、喇叭、耳機、線材轉接 |
| 空氣清淨機、吸塵器、掃地機、電暖器 | 小家電 | 空氣調節、清潔 |
| 吸頂燈、燈具 | 小家電或生活百貨 | 照明 |
| 冰箱、冷凍櫃、洗衣機、冷氣、洗碗機 | 大家電 | 冷藏冷凍、洗衣烘衣、空調、廚房、固定安裝 |
| 米、調味料、洗衣精、物流箱 | 生活百貨 | 食材、日用品、清潔、收納整理 |
| 浴櫃、水龍頭、免治馬桶座、衛浴暖風機 | 生活百貨或小家電 | 衛浴、五金工具、固定安裝 |

既有 62 筆 published 商品的逐筆 mapping 屬實作階段工作，規劃階段只收斂以上方向性判斷。

## 已確認決策

1. **Tag 定位**：以 category 內 subcategory 細分為主、跨 category 共用為輔，方便找商品優先。
2. **廚房降為 tag**：不設「廚房」主分類，以共用 tag 跨小家電、大家電、生活百貨呈現。
3. **新增「照明」tag**：涵蓋吸頂燈等既有商品與未來燈具。
4. **「其他」不配 tag**：不建「待分類」「特殊用途」等無篩選語意的 tag。
5. **低耦合（資料驅動 taxonomy）**：taxonomy 的 SSOT 是 `content/taxonomies/*.json`，程式碼不得 hardcode 分類清單（現況 `app/utils/product-schema.ts` 的 `CATEGORY_IDS` z.enum 需改為以 taxonomy 資料驗證），讓增減分類只需改 content、不需改 code。
6. **Guides / links 暫不擴充**：taxonomy 語意重整不涵蓋 guides 與 links；但既有 5 筆的 `category_ids` / `tag_ids` 引用需在遷移時同步更新以維持 build（含「人體工學」tag 的去留），屬實作階段細節。

## 待確認事項

1. `category_id` 是否維持單一主分類，不改資料模型，只透過 `tag_ids` 表達多重語意（建議維持，spec 時定案）。
2. 衛浴帶電設備（免治馬桶座、暖風機）主分類統一放生活百貨，或放小家電（可留到實作階段逐筆判定，但 spec 應定下判斷規則）。
